1)таблица yy_tables:

col_names_nums: массив текущих номеров полей разных типов для текущей таблицы (в формате json_encode())	
1.  возможные типы данных:
  1.1 bool (tinyint(1) (0 или 1))
         дополнительное поле bool_nameXXX - название поля
		 
	если XXX==1, то название полей просто bool, bool_name
  1.2 int (integer)	
         дополнительные поля int_nameXXX - название поля
			int_minXXX - минимальное значение поля
			int_maxXXX - максимальное значение поля
			если минимальное или максимальное значение не указывается, то оно не применяется и не записывается
			  в col_settings
         дополнительное поле width - ширина на экране (если указывается)
  1.3 string (text)
         дополнительное поле width - ширина на экране
		 если указано дополнительное поле height - высота на экране, то это многострочное поле
  
2. также для каждого поля указывается, может ли оно быть пустым (поле bool_notnullXXX)


3. настройки полей (кроме самого значения поля) хранятся в поле таблицы col_settings в виде массива кодированного
     json_decode():

['bool' => [2 /* относится к полю bool2 */ => ['name' => 'Битовое поле', 'not_null' => 1, 'default' => 0]],
'int' => [1 => ['name' => 'Число', 'not_null' => 1, 'default' => 0, 'min' => -9999, 'max' => 9999]]
] 

-----
2) таблица yy_id_col_types:
   (1, 'bool'), (2, 'int'), (3, 'string')

-----
3) таблица yy_tables_col_order - порядок следования столбцов при отображении и вводе данных в таблицу

записи вида (int id, int table_id, int order, int id_col_type /* по списку из таблицы yy_id_col_types */, col_num_or_table_id)
   если id_col_type == 0, то col_num_or_table_id - id вложенной таблицы

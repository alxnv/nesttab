- системный подход
- возможность использовать при проектировании структуры БД большее количество вариантов по сравнению 
    с ЦМС или другими системами.
      например, можно создавать вложенные структуры различных типов (например, дерево элементов, и для каждого
       элемента 3 соответствующих списка)	  
- можно создавать любые индексы для таблиц БД (например, с помощью phpmyadmin)
- для множественных списков изображений можно добавить дополнительные поля для каждого изображения
- вы можете посмотреть имена таблиц и их полей, с тем чтобы в дальнейшем использовать их для доступа к БД в своем коде
- возможно загружать и присоединять к БД документы и изображения. количество производных от исходного изображений - до
    4-х
- максимальная вложенность дерева элементов - до 6
